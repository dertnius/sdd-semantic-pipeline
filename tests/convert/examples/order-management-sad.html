ARCH : Order Management Service — SAD

Created by Jane Smith, last modified on Mar 02, 2024
Sibling page 1
Sibling page 2

The Order Management Service (OMS) owns the checkout path. It favours at-least-once delivery and an POST /orders entry point. The original design used a synchronous call; it is now event-driven. Throughput grows roughly as n2 with fan-out, and the CO2 budget is tracked per region. (internal only)

This page is critical and was reviewed on March 1, 2024. It keeps the latency budget and the request—response contract in sync. See the external runbook, jump to the Data Model below, owned by Jane Smith . A core goal is resilience under partial failure.

2. Operational notes

Scope

This document covers the checkout path only.

Assumption

The cache is warm before peak traffic.

Hot path

Do not retry non-idempotent writes.

Tip

Enable the circuit breaker in staging first.

Design Constraints

Single region of write authority; cross-region reads are eventually consistent.
